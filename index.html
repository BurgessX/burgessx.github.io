XML学习笔记

简介

XML 指可扩展标记语言（eXtensible Markup Language）。

XML 是一种很像HTML的标记语言。

XML 被设计用来传输和存储数据，而 HTML 被设计用来显示数据。

XML 很重要，也很容易学习。

XML 被设计为具有自我描述性。

XML 是 W3C 的推荐标准。

XML 语言没有预定义的标签。

HTML 中使用的标签都是预定义的。HTML 文档只能使用在 HTML 标准中定义过的标签（如

、

等等）。

语法

  • XML 文档必须包含根元素。该元素是所有其他元素的父元素。

    XML 文档中的元素形成了一棵文档树。这棵树从根部开始，并扩展到树的最底端。

  • 所有的元素都可以有子元素

  • XML 声明文件的可选部分，如果存在需要放在文档的第一行

  • 所有的 XML 元素都必须有一个关闭标签

    在 HTML 中，某些元素不必有一个关闭标签，比如：

  • XML 标签对大小写敏感

  • XML 属性值必须加引号（单引号、双引号均可）

  • < 和 & 非法，可用“实体引用”解决

    实体引用 符号 含义
    &lt; < less than
    &gt; > greater than
    &amp; & ampersand
    &apos; ‘ apostrophe
    &quot; “ quotation mark

    注：在 XML 中，只有字符 “<” 和 “&” 确实是非法的。大于号是合法的，但是用实体引用来代替它是一个好习惯。

  • 在 XML 中，文档中的空格不会被删减。

  • XML 以 LF 存储换行。

    在 Windows 应用程序中，换行通常以一对字符来存储：回车符（CR）和换行符（LF）。

    在 Unix 和 Mac OSX 中，使用 LF 来存储新行。

    在旧的 Mac 系统中，使用 CR 来存储新行。

  • XML 元素必须遵循以下命名规则：

    • 名称可以包含字母、数字以及其他的字符
    • 名称不能以数字或者标点符号开始
    • 名称不能以字母 xml（或者 XML、Xml 等等）开始
    • 名称不能包含空格
    • 可使用任何名称，没有保留的字词。

CSS学习笔记

简介

  • CSS 指层叠样式表 (Cascading Style Sheets)

  • 把样式添加到 HTML 4.0 中，是为了解决内容与表现分离的问题

语法

CSS 规则由两个主要的部分构成：

  • 选择器：需要改变样式的 HTML 元素（一个或多个）
  • 声明（一条或多条）：由一个属性和一个值组成

或

冒号和分号后面可以有0个或多个空格。

选择器

CSS 选择器用于“查找”（或选取）要设置样式的 HTML 元素。

我们可以将 CSS 选择器分为五类：

  • 简单选择器（根据元素名、ID 或类来选取元素）
  • 组合器选择器（根据它们之间的特定关系来选取元素）
  • 伪类选择器（根据特定状态选取元素）
  • 伪元素选择器（选取元素的一部分并设置其样式）
  • 属性选择器（根据属性或属性值来选取元素）

不管是哪种类型的选择器，最终都是为了选择 HTML 元素的。

简介

JavaScript 是 web 开发人员必须学习的 3 门语言中的一门：

  1. HTML 定义了网页的内容

JavaScript 是脚本语言，浏览器会在读取代码时，逐行地执行脚本代码。而对于传统编程来说，会在执行前对所有代码进行编译。

用 scanf、gets 与 fgets 输入字符串的区别

结论：

  • 用 scanf() 输入字符串，缓冲区会留有换行符 \n（ASCII 码为 10）
  • 用 gets() 输入字符串，缓冲区的换行符会被清掉
  • 上述二者都不会在数组中存储换行符 \n。二者的字符串末尾都为结束符 \0（即 0x0，ASCII 码为 0）
  • fgets() 会将 \n 也存入数组，结束符也为 \0
  • scanf() 遇到空格、回车和 Tab 都会认为输入结束，所以它不能接收空格，而其他两个可以

浏览器的基本功能：

  • DOM（Document Object Model）渲染引擎

    • HTML 增删改查 API，如 document.createElement(“div”)
    • HTML 数据格式
    • 渲染 Canvas

  • BOM（Brower Object Model）

    • 浏览器实现的一些类，如 Location、Navigator
    • 方法：openssl 提供的 Base64（btoa 等）、AES、DES、MD5

  • JS 引擎

    JS 是一种解释性语言，但官方没有提供解释器，只制定了标准（ECMAScript），而 JS 引擎就是其他公司提供的 JS 解释器

    • 类型：谷歌 V8，微软查克拉， ffg 的 quickjs
    • JSON
